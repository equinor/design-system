## Usage

`SideSheet` is surfaces containing supplementary content.

<ul>
    <li>Side sheets contain content that supplements the screenâ€™s primary UI region.</li>
    <li>Side sheets display a wide variety of content and layouts.</li>
    <li>Side sheets can be visible or hidden based on screen size or user need, and they often become bottom sheets on mobile.</li>
</ul>

```tsx
import { SideSheet } from '@equinor/eds-core-react'

<SideSheet>
  // your content here
</SideSheet>
```

## Examples

### Placement

To open `SideSheet` press <Icon name="filter_list" size={18} color="#007079" /> icon.

<ul>
  <li>Side sheets are anchored to the left or right edge of the screen.</li>
  <li>Side sheets are a fixed width and typically span the height of the screen.</li>
  <li>Side sheets can display a wide variety of content and layouts, ranging from a list of actions to supplemental content in a tabular layout.</li>
</ul>

<Story id="surfaces-sidesheet--placement" />

### Scrollable

To open `SideSheet` press <Icon name="details" size={18} color="#007079" /> icon.

<ul>
  <li>Side sheets can be vertically scrolled independent of the rest of the UI.</li>
  <li>Side sheets scroll position allows contents to persist while the page is scrolled, and vice versa.</li>
  <li>Side sheets cannot be horizontally scrolled.</li>
</ul>
