## Usage

Tables display information in an easy to scan format.

<ul>
    <li>Use tables if you are displaying information or data that will require minimal interaction such as editing or sorting.</li>
    <li>Consider using a <a href="https://zeroheight.com/0b0c666ab/v/latest/p/65c95b-table-data-grid/b/398446">table data grid</a> if the table is using advanced functionalities.</li>
    <li>Use tables to let users compare information in rows and columns.</li>
</ul>

<strong>Table Caption</strong>
<ul>
    <li>Describes the whole content of a table.</li>
    <li>Helps users find, navigate and understand tables.</li>
</ul>

<strong>Table Header</strong>
<ul>
    <li>Tells users what the rows and columns represent.</li>
    <li>The header row can have icons and text.</li>
    <li>Icons can be used for sorting columns as well as a alternative text.</li>
</ul>

<strong>Table Cell</strong>
<ul>
    <li>The cell row can have icons, text, links, inputs, numbers, monospaced numbers and other custom content (by using the placeholder).</li>
</ul>

```tsx
import { Table } from '@equinor/eds-core-react'

const Demo = () => {
    return (
        <Table>
        <Table.Caption>
            <Typography variant="h2">Fruits cost price</Typography>
        </Table.Caption>
        <Table.Head>
            <Table.Row>
                <Table.Cell>Item no.</Table.Cell>
                <Table.Cell>Name</Table.Cell>
                <Table.Cell>Price</Table.Cell>
            </Table.Row>
        </Table.Head>
        <Table.Body>
            <Table.Row>
                <Table.Cell>123-456</Table.Cell>
                <Table.Cell>Kiwi</Table.Cell>
                <Table.Cell>1.5</Table.Cell>
            </Table.Row>
            <Table.Row>
                <Table.Cell>789-012</Table.Cell>
                <Table.Cell>Apple</Table.Cell>
                <Table.Cell>0.5</Table.Cell>
            </Table.Row>
            <Table.Row>
                <Table.Cell>345-678</Table.Cell>
                <Table.Cell>Mango</Table.Cell>
                <Table.Cell>2.5</Table.Cell>
            </Table.Row>
        </Table.Body>
    </Table>
    )
}
```

### Virtual scrolling
Virtual scrolling is a way of displaying large amounts of data in a way that doesn't overload the DOM. 
Instead of setting all the elements in the DOM, only the elements that are visible at any given time is placed in the DOM. When the user scrolls, 
the content of the table is replaced with new elements. 

There are several ways of implementing this, and also a lot of packages available. In the following we will show an example based on code written for us 
by EDS-user <a href="https://github.com/yusijs">Ronnie Laugen</a>, who figured we needed an example of how virtual scroll could be used in the EDS-table.

We added a package called <a href="https://tanstack.com/virtual/v3">react-virtual</a> which solves this problem in a very typical way: 
The table only shows a small subset of the data, and uses an empty row over and under the content to simulate a larger list. To achieve this the heights 
must be recalculated on each scroll. This package provides a hook that does the recalculating and provides the heights and the rows to display in the list.
