## Usage

A `Card` should be easy to scan for relevant and actionable information. <br />
 <ul>
    <li>A card is identifiable as a single, contained unit.</li>
    <li>A card can stand alone, without relying on surrounding elements for context.</li>
    <li>A card cannot merge with another card, or divide into multiple cards.</li>
    <li>Elements, like text and images, should be placed on them in a way that clearly indicates hierarchy.</li>
    <li>Card blocks can be combined in a variety of ways.</li>
  </ul>

  ```tsx
import { Card } from '@equinor/eds-core-react'

<Card>
    <Card.Header>
      <Card.HeaderTitle>
        <Typography variant="h5">Title</Typography>
        <Typography variant="body_short">Sub title</Typography>
      </Card.HeaderTitle>
    </Card.Header>
    <Card.Content>
      <Typography variant="body_short">
         In ornare quam viverra orci sagittis eu volutpat. 
         Vel elit scelerisque mauris pellentesque pulvinar.
      </Typography>
    </Card.Content>
    <Card.Actions alignRight>
      <Button variant="ghost_icon">
        <Icon name="settings" title="settings action" size={iconSize}></Icon>
      </Button>
      <Button variant="ghost_icon">
        <Icon name="save" title="save action" size={iconSize}></Icon>
      </Button>
    </Cardd.Actions>
</Card>
```
