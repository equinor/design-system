## How to Use Beta Components

<br />

### 1. Browse in Storybook

You can explore all EDS 2.0 components right here in Storybook! Components are organized under the **"EDS 2.0"** section in the sidebar.

Each component has:
- ğŸ“– Interactive documentation
- ğŸ® Live playground to test props
- ğŸ¨ Design examples and use cases
- ğŸ” Code snippets for implementation

### 2. Install Beta Version

To actually use these components in your application, install the beta version:

```bash
npm install @equinor/eds-core-react@beta @equinor/eds-tokens
```

### 3. Load Fonts

EDS 2.0 components use two font families: **Inter** for UI text (buttons, inputs, labels) and **Equinor** for headings. Load both via the EDS variable font stylesheet in your HTML `<head>`:

```html
<link rel="stylesheet" href="https://cdn.eds.equinor.com/font/eds-uprights-vf.css" />
```

> **Important:** The older `equinor-font.css` only includes the Equinor font. If you only load that, UI components (Button, TextField, etc.) will fall back to a generic sans-serif because Inter is missing.

### 4. Import CSS

Import component styles separately in your application entry point (this is important for SSR compatibility):

```tsx
import '@equinor/eds-core-react/next/index.css'
```

### 5. Import from `/next` Entry Point

```typescript
import { Button, TextField } from '@equinor/eds-core-react/next'

function MyApp() {
  return (
    <form>
      <TextField label="Project name" />
      <Button>Submit</Button>
    </form>
  )
}
```
