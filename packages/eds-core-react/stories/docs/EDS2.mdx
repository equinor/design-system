# About EDS 2.0 (Beta)

Welcome to the next generation of the Equinor Design System! 🎉

> **Note on naming:** "EDS 2.0" is the design system name; `eds-core-react` is the package version (semver). They are separate; the numbers line up by coincidence. See [Versioning](https://eds.equinor.com/docs/about/about_eds#versioning) for the full explanation.

## What is EDS 2.0?

EDS 2.0 is a complete redesign of our component library, bringing:

- ✨ **Updated design foundations** - New spacing scales, color system, and typography built with CSS variables
- 🎨 **Pure CSS styling** - Moved away from styled-components to vanilla CSS
- 🌗 **Flexible theming** - Support for dark mode and density modes (spacious/comfortable) via `data-*` attributes
- 🎯 **Enhanced accessibility** - Improved WCAG 2.1 AA compliance across all components
- 🚀 **Better performance** - Optimized with modern React patterns and reduced bundle size
- 🔧 **Refined APIs** - Cleaner, more intuitive component interfaces

<br />
## Current Status: Beta

All components in the **"EDS 2.0"** section are currently in **beta testing**. This means:

- ⚠️ **Breaking changes may occur** - APIs are not yet finalized
- 🧪 **Testing phase** - We're gathering feedback and iterating
- 📝 **Documentation evolving** - Stories and examples are being refined
- 🎨 **Design alignment ongoing** - Ensuring perfect match with Figma

## How to Use Beta Components

<br />

### 1. Browse in Storybook

You can explore all EDS 2.0 components right here in Storybook! Components are organized under the **"EDS 2.0"** section in the sidebar.

Each component has:
- 📖 Interactive documentation
- 🎮 Live playground to test props
- 🎨 Design examples and use cases
- 🔍 Code snippets for implementation

### 2. Install Beta Version

To actually use these components in your application, install the beta version:

```bash
npm install @equinor/eds-core-react@beta
```

### 3. Load Fonts

EDS 2.0 components use two font families: **Inter** for UI text (buttons, inputs, labels) and **Equinor** for headings. Load both via the EDS variable font stylesheet in your HTML `<head>`:

```html
<link rel="stylesheet" href="https://cdn.eds.equinor.com/font/eds-uprights-vf.css" />
```

> **Important:** The older `equinor-font.css` only includes the Equinor font. If you only load that, UI components (Button, TextField, etc.) will fall back to a generic sans-serif because Inter is missing.

### 4. Import CSS

Import component styles separately in your application entry point (this is important for SSR compatibility):

```tsx
import '@equinor/eds-core-react/next/index.css'
```

### 5. Import from `/next` Entry Point

```typescript
import { Button, TextField } from '@equinor/eds-core-react/next'

function MyApp() {
  return (
    <form>
      <TextField label="Project name" />
      <Button>Submit</Button>
    </form>
  )
}
```

## When Will EDS 2.0 Be Stable?

EDS 2.0 components will graduate to stable in a **single major release (v3.0.0)** when:

- ✅ All components are implemented and tested
- ✅ Design team has approved final implementations
- ✅ Real-world usage has validated the APIs
- ✅ Documentation and migration guides are complete

This approach ensures:
- 🎯 One clear migration point for all users
- 📝 Complete migration guide covering all changes
- 🔄 Simple import path change (no component renaming)
<br />
## Migration Path

When EDS 2.0 graduates to stable:

```typescript
// Before (beta testing):
import { Button } from '@equinor/eds-core-react/next'

// After (v3.0.0 stable):
import { Button } from '@equinor/eds-core-react'
```

That's it! No component name changes, just update the import path.
