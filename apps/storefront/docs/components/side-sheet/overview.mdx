A _side sheet_ is a a toggleable area that is anchored to the edge of the viewport with supplementary content.

<!--One to two sentences as an introduction to the component-->

## Guidelines

<!--List how to use it, sizes, widths, grid interactions, color allowances -->

Side sheets display content that is supplementary to the primary content. Side sheets can display a list of actions, filters, supplemental content and functionality. Content in side sheets can be broken into tabs.

Side sheets are not permanent so they should always have a way to be opened and closed. They are always to be anchored to the left of the viewport's right edge. They are to be full height, but can start under the _top bar_ if it is visible. Side sheets can be elevated over content using `Elevation/Overlay`, or push primary content to share the same elevation.

**Don't**

<!--Bullet point list if needed -->

- Do not use a left-anchored side sheet if using a _Navigational Drawer_.
- Do not change the width of the side sheet.
- Do not remove the close icon.

### Standard side sheet

Primarily used for desktop and tablet. The side sheet comes in multiple sizes: `Small`, `Medium`, `Large`, `X Large`.

### Modal side sheet

Modular side sheets are used for mobile with the addition of a _scrim_.
