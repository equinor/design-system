# Project Settings [​](#project-settings)

Configure your project's profile, preferences, and access controls.

## Project Details [​](#project-details)

Navigate to **Project Settings** in the sidebar to manage your project's basic information.

| Setting | Description |
| --- | --- |
| **Name** | Project display name |
| **Slug** | URL-friendly identifier (lowercase letters, numbers, and hyphens only) |
| **Description** | Brief description of the project |
| **Visibility** | Toggle to make a project public (visible to everyone) or private (visible to members only) |
| **Base Permission** | Controls implicit access for workspace members without explicit project invites. Options: **Invite only**, **Read**, or **Write**. See [Roles and Permissions](./../team/roles-and-permissions.html#base-permissions) for details. |

## Members [​](#members)

Add and manage project-specific access from the **Members** page in the sidebar:

-   Add workspace members to the project
-   Assign project roles (Owner, Admin, Editor, Viewer)
-   Remove project access

![Project members](/images/settings/project-members-light.png)![Project members](/images/settings/project-members-dark.png)

## Archive Project [​](#archive-project)

At the bottom of the Project Settings page, the **Archive Project** section lets you archive a project. Archiving removes the project from the active list but preserves all tokens, token sets, and branches.

![Project settings archive project](/images/settings/project-settings-archive-project-light.png)![Project settings archive project](/images/settings/project-settings-archive-project-dark.png)
